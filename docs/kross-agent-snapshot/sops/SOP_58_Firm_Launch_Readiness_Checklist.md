# SOP 58: Firm Launch Readiness Checklist

Version: 0.1
Owner: Tax Mogul Bureau
Applies to: Kross Agent for Firms
Phase: 9 - Governance And Launch

## Purpose

Define what must be ready before Kross Agent is launched inside a firm.

## Core Rule

Kross should not go live until access, guardrails, memory scope, and approval flows are verified.

## Launch Checklist

- firm workspace exists
- firm users and roles are configured
- Kross agent email exists
- Kross has approved portal access
- GHL subaccount/API access is connected
- TaxSlayer access is confirmed
- meeting recorder transcript access is confirmed
- SBTPG or Refund Advantage reporting access is confirmed, if applicable
- AI/model provider access is confirmed for the firm
- provider account owner/payer and usage/cost owner are documented
- firm knowledge base has baseline SOPs
- client visibility rules are tested
- approval flows are tested
- audit logs are working
- test client workflow completed

## Acceptance Criteria

This SOP is working when a firm can pass launch review before real client data is processed by Kross.
