# SOP 60: Returns And Revenues Office Pilot Integration

Version: 0.1
Owner: Tax Mogul Bureau
Applies to: Kross Agent for Firms
Phase: 9 - Governance And Launch

## Purpose

Define how Kross Agent should be piloted inside Returns and Revenues Office before broader rollout.

## Core Rule

Returns and Revenues Office should be treated as the first controlled pilot firm, not an unrestricted production launch.

## Pilot Setup

Before pilot:

- confirm firm workspace
- confirm firm owner/admin
- confirm preparers and roles
- create Kross agentmail.to email
- connect GHL subaccount/API
- connect approved TaxSlayer access
- connect meeting recorder transcript source
- connect SBTPG or Refund Advantage reporting access if used
- upload baseline firm SOPs
- create test/demo clients

## Pilot Workflow

1. Run one demo client through transcript intake.
2. Run one demo client through organizer completion.
3. Run one demo client through missing document request drafting.
4. Run one demo client through TaxSlayer staging/handoff.
5. Run one weekly payroll report test, if applicable.
6. Review audit logs.
7. Fix SOP or system issues.
8. Approve controlled live rollout.

## What Kross Must Not Do During Pilot

Kross must not process broad live client data, send unapproved messages, file returns, change bank product settings, or skip audit review.

## Acceptance Criteria

This SOP is working when Returns and Revenues Office validates the end-to-end Kross operating flow before launch.
