# SOP 56: Client Memory Update After Every Action

Version: 0.1
Owner: Tax Mogul Bureau
Applies to: Kross Agent for Firms
Phase: 9 - Governance And Launch

## Purpose

Define how Kross updates client memory, timeline, tasks, and audit history after operational actions.

## Core Rule

Every meaningful Kross action should leave a clean record in the correct client profile.

## Actions That May Require Memory Or Timeline Updates

- transcript imported
- call summarized
- missing document identified
- client message drafted or sent
- organizer completed
- data entry started
- data entry exception created
- sent to preparer review
- quote/invoice drafted
- payroll/report note created

## Workflow

1. Confirm firm/client.
2. Identify action type.
3. Decide whether action updates memory, timeline, task list, or audit log.
4. Draft memory update if needed.
5. Require human approval for official memory changes.
6. Save approved update.
7. Log action.

## Acceptance Criteria

This SOP is working when Kross activity becomes traceable client context without mixing firms or clients.
