# SOP 43: Dependent And Filing Status Entry

Version: 0.1
Owner: Tax Mogul Bureau
Applies to: Kross Agent for Firms
Phase: 7 - TaxSlayer Data Entry

## Purpose

Define how Kross handles filing status and dependent entry.

## Core Rule

Kross may enter confirmed facts, but eligibility and final filing status decisions require preparer review.

## Data Kross May Enter

- dependent name
- dependent SSN/ITIN from secure source
- dependent date of birth
- relationship
- months lived with taxpayer, if provided
- student/disabled indicators, if provided
- filing status selected by client or preparer

## Must Escalate

Kross must escalate:

- custody disputes
- duplicate dependent claims
- unclear support/residency
- married/separated/head of household uncertainty
- missing dependent SSN/ITIN
- conflicting organizer answers

## Acceptance Criteria

This SOP is working when Kross enters provided dependent facts and flags eligibility questions for the preparer.
